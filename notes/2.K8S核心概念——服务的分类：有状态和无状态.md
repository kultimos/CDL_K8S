# 服务的分类

  ## 有状态应用
  - 会有本地环境产生依赖,例如:需要存储数据到本地磁盘
  
  ### 有状态应用的优点
  - 对客户端透明,无依赖关系,可以高效实现扩容、迁移

  ### 有状态应用的缺点
  - 不能存储数据,需要额外的数据服务支撑

  ### 有状态应用代表
  - mysql
  - redis


  ## 无状态应用
  - 不会对本地环境产生任何依赖,例如:不会存储数据到本地磁盘

  ### 无状态应用的优点
  - 可以独立存储数据,实现数据管理

  ### 无状态应用的缺点
  - 集群环境下需要实现主从、数据同步、数据备份、水平扩容复杂

  ### 无状态应用代表
  - nginx
  - apache
